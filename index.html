Horse Racing Predictor

Project objective: Train a machine learning model to predict the outcome of horse races.

Horse Racing Predictor

Motivation

Inspiration

My first step into the world of gambling was through matched betting - a no-risk methodology which uses Bookmaker offers to guarantee profits. This was what sparked my interest in betting systems and technically-driven approaches to gambling.

Le Wagon's Data Science bootcamp connected me with other people who were interested in sports betting algorithms, and my interest was piqued once again.

Advantages of Machine Learning Methods

As gambling can be addictive, it is very easy to become emotionally involved and make bad decisions. A robo-betting strategy provided by a machine learning model removes the emotional decision making. The bettor does not have to review racecards or analyse horses, nor do they need to analyse their wins and losses.

Although emotions are not completely removed, a machine learning driven methodology allows the bettor to focus on the long-term and not be swayed by short-term losses.

Project Overview

Objectives

Create a model that is able to predict the outcome of horse races.

Develop a betting strategy that can leverage the model's predictions to make a profit.

Data Collection

Horse racing data is difficult to find, with only a few patchy datasets on Kaggle available.

Searching Github, I came across RPScrape, a utility for scraping results from Racing Post. From this I gathered around 1.5 million datapoints for use in exploratory data analysis model training.

Using this, I have built automated systems which will scrape results daily, as up-to-date data is crucial for accurate model predictions.

Data Processing

Horse racing data in isolation is almost useless, with no features correlating with a horse winning. To begin making sense of all the data, it requires history and context.

I engineered various features using Pandas built-in aggregation utilities, including aggregations of horse's performance over its lifetime, alongside comparisons with its competitors within a race.

Model Training

As horse racing is notoriously difficult to predict with any consistency, machine learning models suffer likewise, with overfitting being a major issue.

My benchmark for a model was defined by by running a simulation which randomly picked one horse from every race, resulting in a 13% success rate.

I have trained two separate models, a deep neural network and a random forest classifier, which are both able to capture complex relationships within data.

By carefully balancing parameters, the neural network is able to predict winners with around 25% Precision, which is a mild success when compared to the benchmark.

My second model, using ensemble machine learning algorithms, allowed my model to hit around 60% precision predicting winners, but at the cost of providing a winning candidate only once a week on average.

Conclusion

Does It Make Money? The Short Answer

Using a well defined betting strategy, my model is able to make a profit over time.

Does It Make Money? The Long Answer

Whilst trialling betting strategies over my dataset, I found that merely randomly picking a horse from each race can occassionally result in enormous profits (although most of the time it results in a loss).

There is no way to tell if the betting strategy is going to be successful forever, it is merely that confidence increases the longer it is profitable.

Personal Growth and Lessons Learnt

This project was a huge exercise in taking data with almost no meaningful features and turning it into something useful.

Upon completing my first model, which took me around a month, I found the results to be disappointing. At this point, it would have been easy to give up and move on, but I instead tried to further my domain knowledge in horse racing, which lead to reading about the Bayer Speed Figures used in Hong Kong races. Inspired by this I created my own speed ratings to benchmark horses against, and this proved to be a crucial piece of the puzzle.

This project is also the first time I have ensembled multiple models in order to play on the strengths of both. This works well with the domain of horse racing, as the difficulty in making consistent predictions means both models may complement each other. If one model performs poorly one day, the second model may be there to make up for it, and using results that both models agree on can be a good indicator of a strong candidate.

The Future
Automation

This project is already automated to the point where a single Bash script will gather results and racecards, then perform data processing and modelling and finally email the relevant results to me so I have access to them remotely.

This may be further automated with use of the cron utility and a system such as a Raspberry Pi to host the service.

While bets are currently manually placed, Betfair Exchange has an API which can be utilised to automate betting.

Reddit Sentiment Analysis

Motivation

Project Outline

This project was our final group project at the Le Wagon Data Science bootcamp.

Reddit is a social media platform where users can post content and interact with other users. It is a popular platform for discussion, with many subreddits dedicated to specific topics.

The project outline was to analyse the sentiment and emotion of Reddit comments, specifically those posted on the infamous Wall Street Bets subreddit, which is dedicated to stock market discussion.

The project was inspired by the recent GameStop short squeeze, which was driven by the Wall Street Bets subreddit.

Project Overview

Objectives

Create a system for collecting Reddit comments, then analyse the sentiment and emotion of the comments using Natural Language Processing (NLP).

Present the results in a dashboard, which can be used to track the sentiment and emotion of the subreddit over time.

Under the Hood

The backend for the project for data collection and processing was written in Python, using the PRAW library for interacting with the Reddit API.

Modelling was performed using transformers, a library for NLP tasks.

The dashboard was created using Streamlit, a Python library for creating interactive dashboards.

An API was created using FastAPI to allow the dashboard to interact with the backend.

Conclusion

Personal Growth and Lessons Learnt

This was my first experience of working on a group project, and aside from developing technical skills, it was a great opportunity to develop soft skills such as communication and teamwork.

This was also my first experience of working on a live product, and as such I worked closely with deployment tools such as Docker and Google Cloud Platform services.

Furthermore, this project introduced me to the practical side of data science, demonstrating how technical proficiency intertwines with user experience, performance optimisation, and scalability.

The challenges of creating a live, dynamic project honed my problem-solving skills and deepened my understanding of cloud-based solutions.

The Future
Further Development Potential

As this was a short lived project during a part time bootcamp, there is plenty of scope for further development.

An automated data processing system proved to be a challenge, as the operations were very intensive and time consuming, meaning running this on Google Cloud Run was not feasible.

The project could also benefit from utilising a cloud based database system such as Google BigQuery, which would allow for more efficient data storage and retrieval and allow the dashboard to be updated in real time.

To enhance scalability and automation, Kubernetes could be explored for container orchestration. Leveraging Kubernetes would not only ensure the smooth operation of the project but also facilitate fully automated processes, reducing the need for manual intervention.

Codewars Web Scraper and Data Visualisation

Motivation

Inspiration

I have been using coding problems to further my technical capabilities since I started learning Python and also use them as a way to explore new languages and paradigms.

My platform of choice is Codewars, and as I racked up solutions I wanted a way to be able to extract them all.

As I was learning new concepts with Le Wagon, I decided to use this as an opportunity to build a web scraper and data visualisation project with my newly learnt skills.

Project Overview

Project Objectives

The primary goal of this project is to build a customised web scraper for retrieving Codewars problem solutions and create an interactive data visualisation notebook to accompany the problems.

Data Collection

Initially I built a static webscraper, requiring me to preload the page with the number of problems I wanted to scrape and save it locally. Codewars loads solutions dynamically, so as I solved more problems, this became more cumbersome.

I later updated the scraper to use Selenium to login to Codewars using a headless browser and simulate scrolling to the bottom of the page. Beautiful Soup then parses all the problems, which are saved locally and organised into folders based on the problem's programming language.

Data Visualisation

Instead of just leaving these scraped problems to sit archived on my hard disk, I decided to bring it to life by creating a Jupyter Notebook to visualise the data and explore interesting features.

By employing libraries such as Matplotlib and Seaborn, I'm able to visualise my problem solving behaviour over time.

For example, I can see how many problems I solved each day, how many problems I solved in each language, and how many problems I solved in each difficulty.

I also use the Codewars API to collect additional information about problems, which allows me to uncover novel insights such as the 'rarest kata', the Codewars problem I've solved that has the least solved, or the 'most difficult kata'; the Codewars problem that has the lowest rate of completion.

Impact and Future Steps

This project forms the foundation for future explorations in coding challenges and data analysis.

The Jupyter Notebook can handle problems dynamically, meaning it is self-updating as I solve more problems.

Conclusion

My Codewars Web Scraper and Data Visualisation project shows my passion for coding, data exploration, and problem-solving.

The Jupyter Notebook is a great way to visualise the progress I've made and I look forward to continuing to use it as I solve more problems.

Further Development

The Codewars scraper contains a Python dictionary which matches programming languages to file extensions and comment syntax. This only contains the programming languages I have used on Codewars and may be extended to contain details for all programming languages Codewars offers.

Other Minds

Motivation

Inspiration

Other Minds draws its roots from philosophy, specifically looking towards the question; why are we ourselves, and why is understanding the consciousness of others so difficult? This is a concept in philosophy aptly knows as the problem of other minds - how do you know others experience consciousness?

Other Minds is an exploration of this concept. You inhabit a character known as Ed, accompanied by a floating prescence called Sage who allows you to possess other entities in the game. Through this unique mechanic, the player can step into the shoes of other characters, use them to solve puzzles and advance the narrative of the game.

Other Minds draw influence from Seiklus , a cornerstone of Gamemaker's legacy. By combining the influence of this game with innovative gameplay mechanics, Other Minds is a game which allows the player to explore the concepts of consciousness and embark on an exploratary adventure.

Project Overview

Modular Design

Other Minds is an expansive, open world 2d platformer. Modular, re-usable code was essential for streamlining the development process and allowing the game to be both scalable and amenable to the addition of new features.

As the player is free to control any character, their abilities are determined by variables dictating speed, acceleration and jump height, alongside a list of boolean variables to determine which skills are accessible. Using a finite state machine, both the main character and various other characters, such as a bird, run on the same code, demonstrating how powerful this concept is.

Character AI

Character movement should be consistent whether the character has 'free will' or if the player is controlling them. To accomplish this, characters use simulated key presses to create movement. The game world is full of characters who are able to exhibit their unique behaviours and personalities, be it slimes who have fun jumping up and down, or a nervous character who paces back and forth. The use of simulating movement as if the player were controlling them allows this.

This concept can allow for extremely complex emergent behaviour for characters. Of note is the first boss fight of the game where the player faces antagonist Dr Death. Upon defeating his machines, Dr Death himself runs into the boss arena to try and attack you. He is able to determine which platforms he can reach and calculate which is best for reaching his target; the player. A few simple rules allows the AI to become very complex.

Conclusion

Personal Growth and Lessons Learnt

Other Minds was my first large project after a long programming hiatus. Whilst this project does have somewhat organised code, my coding ability, code modularity and project management vastly improved over the course of the development process. This improvement is demonstrated in project such as Ben Was Assimilated, which made the process of adding new enemies or items to the game a case of adding one line of code, with the modular systems managing everything else.

The game also required careful data management, for which I made use of bitmasking and data structures. Bitmasking is a powerful concept which I have used in almost every project since.

The development process took place over the course of several years, and maintaining a project for that long can be difficult in terms of motivation. This taught me skills in large project management; maintaining an organised task list and breaking steps down into small, manageable chunks.

The Future

Other Minds has an unknown future as a concept and no more work will be done on the game itself, however I have often considered making a sequel, or re-imagining, since my abilities have grown hugely since I first embarked on making this game. I would be able to combine everything I have learnt over the past few years to create a more immersive experience, with improved game mechanics and storytelling.

The one large barrier is time - producing a project this large as a solo developer can be extremely consuming, especially when you are working fulltime alongside.

Date and Game

Project objective: Create a framework for 2-player online games to integrate into a dating app.

Date and Game

Project objective: Create a framework for 2-player online games to integrate into a dating app.

Date and Game

Motivation

Inspiration

Date & Game is a company integrating online games and dating into one app. Users match with other users and connect to play games together.

I was the lead games programmer for the company, in charge of creating the online game framework to integrate with the Date & Game API and programming gameplay.

I collaborated with the app developers for API integration and worked alongside game designers and artists to create bespoke, two player experiences.

Project Overview

Online Games

I produced a robust framework using Gamemaker Studio 2 and Socket IO as the basis for all games.

I created a variety of different games from Route 69, a game where players go on an adventure together, collecting key items to solve puzzles, to PixlWave, a vaporwave stylised game where players compete to complete pixel art drawings as quickly and accurately as possible.

Framework

To streamline the game development pipeline, I create an online game framework with a high level of modularity, allowing for the creation of new games with minimal effort. The basic framework allows players to send position and other variable updates and also keeps the game's world state synchronised, allowing for arbitrary resynchronisation - for example, if one player leaves the game, they are able to return and have the game state resynchronise so both players see the game in the same state.

Technical Details

Online games pose their own unique problems due to latency. For example, one player may pick up an item, then the other player may pick up the same item before the signal that the item has been picked up. These sort of conflicts arise constantly and methods must be implemented for clean resolution.

I implemented various techniques to ensure the robustness of the framework and to optimise performance. This involved the use of three way handshakes, exponential backoff and data-encoding.

Data management was crucial, due to the requirement of arbitrary resychronisation. Data structures were put in place along with encoding methods to allow players to seamlessly send large amounts of data easily and only where necessary.

Conclusion

Personal Growth and Lessons Learnt

Despite initially having no experience with online game development and having a steep learning curve before me, I created robust systems and used creative problem solving to resolve any conflicts that arise as a result of latency to ensure smooth synchronisation of the games I developed.

I also transferred the systems from Gamemaker Studio 2 to Unity, a system for which I previously had little familiarity with.

Personal Homepage

Motivation

Inspiration

Over the years I've worked on a lot of games, pieces of music, and other projects, and I wanted a place to host and write about these projects.

I have deep sense of nostalgia for the Web 1.0 era, a time when hosts like Freewebs and Geocities proliferated. These websites had a charming, amateur quality with a distinct personal touch which left an impression on the viewer.

Inspired by this, I chose to use web hosting service Neocities , the spiritual successor to Geocities.

Project Overview

Neocities

Neocities is a free web hosting service that allows users to create their own websites. It is a successor to Geocities, a web hosting service that was popular in the 90s and early 2000s.

When a user creates a site on Neocities, they are provided with one file, index.html.

From there, the user can create more files and folders to build up a website. These were the first steps I took into front end web development, creating a simple and bold site, with minimal javascript to maximise compatability.

Tools

As my site grew, I developed tools in Python to dynamically generate pages for me. For example, my games pages contains over 30 games with a variety of tags and CSS-based filtering. To create or update this page, I run a Python script which will automatically parse json data and generate html and accompanying css.

Chronologically updated pages such as my blog also have accompanying tools to automate building pages and updating links and RSS feeds.

Neocities API

As my site grew, uploading pages manually became cumbersome, so I have built file-tracking and automation tools that use the Neocities API to update any pages as necessary. Site updates are now as simple as running a Python script.

Conclusion

Personal Growth and Lessons Learnt

Neocities was my first foray into front end web development, and was the cornerstone of my interest in web development in general. From here I have built up the website and accompanying automation tools to streamline the process of web development and deployment, and this has been the foundation I needed to continue working with web development projects.
