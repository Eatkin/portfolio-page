Horse Racing Predictor

Project objective: Train a machine learning model to predict the outcome of horse races.

Horse Racing Predictor

Motivation

Inspiration

My first step into the world of gambling was through matched betting - a no-risk methodology which uses Bookmaker offers to guarantee profits. This was what sparked my interest in betting systems and technically-driven approaches to gambling.

Le Wagon's Data Science bootcamp connected me with other people who were interested in sports betting algorithms, and my interest was piqued once again.

Advantages of Machine Learning Methods

As gambling can be addictive, it is very easy to become emotionally involved and make bad decisions. A robo-betting strategy provided by a machine learning model removes the emotional decision making. The bettor does not have to review racecards or analyse horses, nor do they need to analyse their wins and losses.

Although emotions are not completely removed, a machine learning driven methodology allows the bettor to focus on the long-term and not be swayed by short-term losses.

Project Overview

Objectives

Create a model that is able to predict the outcome of horse races.

Develop a betting strategy that can leverage the model's predictions to make a profit.

Data Collection

Horse racing data is difficult to find, with only a few patchy datasets on Kaggle available.

Searching Github, I came across RPScrape, a utility for scraping results from Racing Post. From this I gathered around 1.5 million datapoints for use in exploratory data analysis model training.

Using this, I have built automated systems which will scrape results daily, as up-to-date data is crucial for accurate model predictions.

Data Processing

Horse racing data in isolation is almost useless, with no features correlating with a horse winning. To begin making sense of all the data, it requires history and context.

I engineered various features using Pandas built-in aggregation utilities, including aggregations of horse's performance over its lifetime, alongside comparisons with its competitors within a race.

Model Training

As horse racing is notoriously difficult to predict with any consistency, machine learning models suffer likewise, with overfitting being a major issue.

My benchmark for a model was defined by by running a simulation which randomly picked one horse from every race, resulting in a 13% success rate.

I have trained two separate models, a deep neural network and a random forest classifier, which are both able to capture complex relationships within data.

By carefully balancing parameters, the neural network is able to predict winners with around 25% Precision, which is a mild success when compared to the benchmark.

My second model, using ensemble machine learning algorithms, allowed my model to hit around 60% precision predicting winners, but at the cost of providing a winning candidate only once a week on average.

Conclusion

Does It Make Money? The Short Answer

Using a well defined betting strategy, my model is able to make a profit over time.

Does It Make Money? The Long Answer

Whilst trialling betting strategies over my dataset, I found that merely randomly picking a horse from each race can occassionally result in enormous profits (although most of the time it results in a loss).

There is no way to tell if the betting strategy is going to be successful forever, it is merely that confidence increases the longer it is profitable.

Personal Growth and Lessons Learnt

This project was a huge exercise in taking data with almost no meaningful features and turning it into something useful.

Upon completing my first model, which took me around a month, I found the results to be disappointing. At this point, it would have been easy to give up and move on, but I instead tried to further my domain knowledge in horse racing, which lead to reading about the Bayer Speed Figures used in Hong Kong races. Inspired by this I created my own speed ratings to benchmark horses against, and this proved to be a crucial piece of the puzzle.

This project is also the first time I have ensembled multiple models in order to play on the strengths of both. This works well with the domain of horse racing, as the difficulty in making consistent predictions means both models may complement each other. If one model performs poorly one day, the second model may be there to make up for it, and using results that both models agree on can be a good indicator of a strong candidate.

The Future
Automation

This project is already automated to the point where a single Bash script will gather results and racecards, then perform data processing and modelling and finally email the relevant results to me so I have access to them remotely.

This may be further automated with use of the cron utility and a system such as a Raspberry Pi to host the service.

While bets are currently manually placed, Betfair Exchange has an API which can be utilised to automate betting.

Codewars Web Scraper and Data Visualisation

Project objective: Build a Web Scraper for Codewars Problems and Create a Data Visualization Notebook

Codewars Web Scraper and Data Visualisation

Motivation

Inspiration

I have been using coding problems to further my technical capabilities since I started learning Python. I also use them as a way to explore new languages and paradigms.

My platform of choice is Codewars, and as I racked up solutions I wanted a way to be able to extract them all.

As I was learning new concepts with Le Wagon, I decided to use this as an opportunity to build a web scraper and data visualisation project with my newly learnt skills.

Project Overview

Project Objectives

The primary goal of this project is to build a customised web scraper for retrieving Codewars problem solutions and create an interactive data visualisation notebook to accompany the problems.

Data Collection

Initially I built a static webscraper, requiring me to preload the page with the number of problems I wanted to scrape and save it locally. Codewars loads solutions dynamically, so as I solved more problems, this became more cumbersome.

I later updated the scraper to use Selenium to login to Codewars using a headless browser and simulate scrolling to the bottom of the page. Beautiful Soup then parses all the problems, which are saved locally and organised into folders based on the problem's programming language.

Data Visualisation

Instead of just leaving these scraped problems to sit archived on my hard disk, I decided to bring it to life by creating a Jupyter Notebook to visualise the data and explore interesting features.

By employing libraries such as Matplotlib and Seaborn, I'm able to visualise my problem solving behaviour over time.

For example, I can see how many problems I solved each day, how many problems I solved in each language, and how many problems I solved in each difficulty.

I also use the Codewars API to collect additional information about problems, which allows me to uncover novel insights such as the 'rarest kata', the Codewars problem I've solved that has the least solved, or the 'most difficult kata'; the Codewars problem that has the lowest rate of completion.

Impact and Future Steps

This project forms the foundation for future explorations in coding challenges and data analysis.

The Jupyter Notebook can handle problems dynamically, meaning it is self-updating as I solve more problems.

Conclusion

My Codewars Web Scraper and Data Visualisation project shows my passion for coding, data exploration, and problem-solving.

The Jupyter Notebook is a great way to visualise the progress I've made and I look forward to continuing to use it as I solve more problems.

Further Development

The Codewars scraper does not include any error trapping for if the user enters incorrect login credentials or if the scraper is comes across an unexpected programming language, as detailed below.

The Codewars scraper contains a Python dictionary which matches programming languages to file extensions and comment syntax. This only contains the programming languages I have used on Codewars and may be extended to contain details for all programming languages Codewars offers.

Other Minds

Project objective: a 2d platformer where players can take control of any other entity in the game.

Other Minds

Motivation

Inspiration

Other Minds was conceptually influenced by philosophy. Inparticular, why are we ourselves? Why can't we know what it is to be other people? This is a concept in philosophy known as the problem of other minds - how do you know others experience consciousness?

Other Minds is a game that explores this concept. You play as the main character Ed, who is accompanied by a floating green whisp called Sage who can possess other entities in the game. The player can then control these entities, and use their abilities to solve puzzles and progress through the game.

It was heavily influenced by the classic Gamemaker indie game Seiklus.

Project Overview

Modular Design

Other Minds is a relatively large, open world 2d platformer. Modular, re-usable code helped to streamline the development process and made the addition of new features simple.

Since every character is controllable, their abilities are determined by variables dictating things such as speed, acceleration and jump height, alongside a list of boolean variables to determine which abilities are and are not available. Every character runs on the exact same script. Using a finite state machine, both the main character and various other characters, such as a bird, run on the same code, which demonstrates how powerful this concept is.

Character AI

Every character in the game should move in the same way whilst they have 'free will' and whilst the player is controlling them. To accomplish this, characters use simulated key presses to create movement. The game world is supposed to be full of characters who are able to exhibit behaviours, be it slimes who have fun jumping up and down, or a nervous character who paces back and forth. The use of simulating movement as if the player were controlling them allows this.

Using some situational awareness, this can allow for extremely complex emergent behaviour for characters. Of note is the first boss fight of the game where the player faces antagonist Dr Death. Upon defeating his machines, Dr Death himself runs into the boss arena to try and attack you. He is able to determine which platforms he can reach and calculate which is best for reaching his target, the player. The emergent behaviour based on a few simple rules allows the AI to become very complex, and infact, I had to deliberately dumb down the behaviour and create 'mistakes' because the boss fight became way too difficult.

Conclusion

Personal Growth and Lessons Learnt

Other Minds was my first large project after a long break from programming. Whilst this project does have somewhat organised code, my coding ability improved meaning that in future I could bring forward what I learnt. This is demonstrated in project such as Ben Was Assimilated , which made the process of adding new enemies or items to the game a case of adding one line of code, and the modularity of the code did the rest.

The game also required careful management of data, for which I made use of bitmasking and data structures. Bitmasking is a powerful concept which I have used in almost every project since.

This project also took place over the course of several years, and maintaining a project for that long can be difficult in terms of motivation.

The Future

Other Minds has now been shelved and no more work will be done on it. I have often considered making a sequel, or re-imagining, since my abilities have grown hugely since I first embarked on making this game. I would be able to combine everything I have learnt over the past few years to create a more immersive experience, with improved game mechanics and storytelling.

The one large barrier is time - producing a project this large can take it's toll, especially when you are working fulltime alongside.

Date and Game

Project objective: Create a framework for 2-player online games to integrate into a dating app.

Date and Game

Motivation

Inspiration

Date and Game is a company aiming to integrate online games into a dating app. Users can match with other users and connect to play games together.

I was the lead games programmer for the company, in charge of creating the online game framework to integrate with the Date & Game API and programming gameplay.

I collaborated with game designers and artists to create bespoke, two player experiences, and also collaborated with the app developers for API integration.

Project Overview

Online Games

I produced a robust framework using Gamemaker Studio 2 and Socket IO as the basis for all games.

I created a variety of different games from Route 69, a game where players adventure together, collecting key items to solve puzzles, to PixlWave, a vaporwave stylised game where players compete to complete pixel art drawings as quickly and accurately as possible.

Framework

I developed the online game framework to be as modular as possible, allowing for the creation of new games with minimal effort. The basic framework allows players to send position updates and send variable updates. It also keeps the game's world state synchronised, allowing for arbitrary resynchronisation - e.g. if one player leaves the game, they are able to return and have the game state synchronised between players.

The framework also allows arbitrary resychronisation of games. If one player leaves the game, they should be able to return to the game at any point.

Technical Details

I implemented various techniques to ensure the robustness of the framework and to optimise performance. This involved the use of three way handshakes, exponential backoff and data-encoding.

Data management was crucial, due to the requirement of arbitrary resychronisation. Data structures were put in place along with encoding methods to allow players to seamlessly send large amounts of data easily and as necessary.

Online games pose their own unique problems due to latency. For example, one player may pick up an item, then the other player may pick up the same item before the signal that the item has been picked up. These sort of conflicts arise constantly and methods must be implemented for clean resolution.

Conclusion

Personal Growth and Lessons Learnt

I initially had no experience with the creation of online games, so in spite the massive learning curve before me, I created robust systems and used creative problem solving to resolve any conflicts that arise as a result of latency and to ensure smooth synchronisation of the games.

I also transferred the systems from Gamemaker Studio 2 to Unity, a system for which I have little familiarity with.

Personal Homepage

Project objective: Create a personal homepage to host projects I've made.

Personal Homepage

Motivation

Inspiration

I have a lot of nostalgia for the days of Web 1.0, when the internet was more amateur and lots of people made their own websites on hosts like Geocities and Freewebs. These websites were flawed, however, they had a very personal touch.

It was because of this I decided to use web hosting service Neocities, the spiritual successor to Geocities.

My website is a place to host things I have made, including my personal blog and my webcomic.

Project Overview

Neocities

Neocities is a free web hosting service that allows users to create their own websites. It is a successor to Geocities, a web hosting service that was popular in the 90s and early 2000s.

When a user creates a site on Neocities, they are provided with one file, index.html.

From there, the user can create more files and folders to create a website. This was the first steps I took into front end web development, and after many revisions, I have a website I'm happy with. The style is supposed to be simple and bold, with minimal javascript to maximise compatability.

Tools

As my site grew, I began to develop tools using Python to dynamically generate pages for me. For example, my games pages contains over 30 games with a variety of tags and CSS-based filtering. To create or update this page, I run a Python script which will automatically parse json data and generate html and accompanying css.

Chronologically updating pages like my blog or webcomic also have accompanying tools to build pages, update links and create RSS feeds.

Neocities API

As my site grew, uploading pages manually became cumbersome, so I have built file-tracking and automation tools that use the Neocities API to update any pages that have changes. This means updating pages such as my blog become trivial, as I only need to run two scripts to build the pages and update the website.

Conclusion

Personal Growth and Lessons Learnt

Neocities was my first foray into front end web development, and was the first project that took my interest in web development in general. From here I have built up the website and accompanying automation tools to streamline the process of building and updating the website, and have completed several other web development projects since.

DOScord

Project objective: Create a dynamic, DOS-influenced website inspired by Discord.

DOScord

Motivation

Inspiration

DOScord was created as a way to practice dynamic website creation using JavaScript. It aims to emulate the functionality of Discord, but with a fun DOS theme.

Project Overview

Website Structure

DOScord uses the CSS-grid layout to clearly define container areas within the page. This allows for easy, responsive design, and allows for the page to be easily split into sections.

Dynamic Content

As with Discord, DOScord is divided into servers and channels, with each channel containing a number of messages. The content of each channel is dynamically generated using JavaScript. The data is sourced from several linked JSON files, which are parsed and used to generate the content as needed.

Conclusion

Personal Growth and Lessons Learnt

DOScord was a great opportunity to practise working with JavaScript and a complex structure of JSON data. This allowed me to create a dynamic website with a clear structure, which I have been able to bring forward to future projects, such as the use of dynamic content on this website.

Future Plans

DOScord is currently a work in progress, with plans to add more content to the page for users to explore, along with additional features such as user profiles.
